# 原本我们一直专注server模式
原因
1. assembly首次加载非常慢
1. server模式可以直接注入业务逻辑层服务，比前后端分离方式简单太多。

# net8中blazor auto模式
首次打开页面会以server模式运行，后台会默默下载dll，完成时会自动切换到assembly，也就是客户端模式

# 现在
由于net8尚未发布，不过我们可以先用客户端模式跑起来，后面改下配置即可。

原本的代码暂时不动，等前端架子搭起来后再删除

没必要让前端和后端托管到一个进程中了。

## 前后端共享
接口和dto是前后端共享的，所以可以单独定义个项目

## 多应用
每个应用有自己的application层，它们共享common应用
类似的前端分开做，不过它们有部分共享内容，如：对后端接口的请求核心方法
所以也为此定义一个项目
