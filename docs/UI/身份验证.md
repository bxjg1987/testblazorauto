此文档仅描述身份验证中前端部分的处理。

首次访问首保护的页面时，
asp.net core自带的授权中间件通过目标页面的endpoint拿到Authorize定义的授权策略，执行授权检查后，不通过，跳转到登录页
用户输入账号密码，提交
页面的逻辑部分在服务端执行，它从后端api拿token
并使用cookie登录，存储token，然后跳转到受保护的页面
此时授权检查通过，进入页面
server模式运行时，已经是身份验证通过的

通过PersistingRevalidatingAuthenticationStateProvider存储token，
在wasm模式启动时PersistentAuthenticationStateProvider反序列化拿到token，存储在内存中

所以服务端部分是标准的cookie登录，然后通过PersistentComponentState机制将身份验证传递到浏览器中，以便wasm模式能正常运行。
