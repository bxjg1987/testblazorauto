# 如果要在领域服务中触发事件，外部通常应该只是单次调用{#trigger-event-only-in-app-service}
用户的一次请求通常调用某个应用服务方法，内部可能多次调用领域服务的方法，比如一次性添加订单
及其明细时，添加明细的领域服务方法就会被调用多次，若在领域服务方法中触发事件，则一次请求会触发事件多次

还有一种情况是，一个应用服务方法中，做了某个状态的变动，之后的某个调用又做了状态变动
刚好两次变动后，状态变为了最初的状态
所以此时去发事件也是不合适的。

所以如果要在领域服务中触发事件，外部通常应该只是单次调用

# 模块服务注册与配置
模块往往需要注册很多服务，做各种配置。
关于配置，可以使用模块配置类（内存级别的）和abp提供的settings系统；

服务注册，若模块依赖的是接口，在模块内部可以注册默认实现，
若模块依赖的不是接口，而是具体的实现，则也在模块内部注册。

其它模块引用上面说的模块时，若上面的模块是依赖的接口，可以在当前模块内替换，这里替换的也仅仅是一个默认实现，
最终可用在入口项目去替换具体的实现，或引用上面两个默认实现入口项目不做任何配置。
