# token立即失效

在asp.net core identity模块中，用户表有个SecurityStamp字段，
在做敏感操作，如修改密码时，会更新SecurityStamp字段，
如果有额外的字段需要更新此字段，可以调用identity框架的UserManager.UpdateSecurityStampAsync方法。

在默认的cookie身份验证中，会验证此字段，不过它默认有个验证的间隔时间，因为几乎每个请求都会做
身份验证，若每次都查询数据库验证此字段，太浪费了。

但在默认的jwttoken身份验证中，没有验证这个字段，猜测原因是token验证是无状态的。

但我们确实希望改密码，禁用用户时，让之前的token立即失效，


参考：
https://github.com/aspnetboilerplate/aspnetboilerplate/issues/6813
https://github.com/aspnetboilerplate/aspnetboilerplate/issues/5975

2025-3-16 估计abp默认都没有实现这个，因为先登录用户，在不注销情况下修改密码，之前登录的用户登录的token仍然有效，

所以需要我们自己按上面的参考处理下。

注意：如果SecurityStamp没有在token中，使用abp预留的xxfactory在登录时保留这个属性。



# 黑名单方案
若使用黑名单方案，必须持久化黑名单，否则若是内存存储，应用服务重启后，黑名单就起不到作用了。
最好是把黑名单存储在redis中，但这又强制使用redis不好，如果不用redis每次查数据库比对浪费性能。

# SecurityStamp
也是每次查询数据库，浪费性能。搞个缓存，若某一次验证成功，就缓存这个用户5分钟，这期间就不验证了。
