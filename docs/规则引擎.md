有时候同一个规则在前端和后端都需要，比如在订单明细管理时，若订单是未审核状态时，则可以随意更改，否则不允许修改明细。
再比如 根据选择的商品类别和其它因素 结合 折扣规则 计算 最终金额等。

这类规则在前后端都需要做，某些规则可能还有变动，比如不同租户针对同一个业务，规则不同。

针对前后端都需要的规则，针对不同情况规则变动多 的场景，使用规则引擎很方便。


# 返回值可以实现流程

规则引擎除了可以判断真假，还可以通过Action返回数据
所以可以实现简单的流程引擎效果

# 规则的存储
数据库或文本文件都行、硬编码也是可以接受的  
大部分的规则仅仅是为了前后端共享，少部分特殊场景才考虑规则频繁变动
