# abp的事件
它一个是数据库层面，工作单元提交前后会触发事件，这种事件往往是启动配置时注册事件处理器。
另一种是动态的，运行时 注册、注销、触发事件，
这非常好。
但目前，8.3.1版本中，运行时动态注册事件会报线程安全错误，就是注册事件的同时，可能有事件在触发，abp没有处理线程安全问题，此错误规划到下一个版本会修复。

# MediatorR
这是个轻量级的事件总线库

# Zhongjie

我们自己实现的，它是个委托容器，注册事件时加委托进去，触发时遍历并执行里面的委托，它是线程安全的。

## 事件范围
就本项目而言

整个应用程序范围、
租户范围、
当前用户
当前请求（主要应用于blazor server）

在内层注册，包括自己的和逐层向外的，委托存储在自己所在层
那么注销也一样

应用层 应用的字符串为空
租户_子应用_用户

在应用层基础上 加个 级联值，代表 某个blazor应用，所有当前界面中的事件都可以使用这个范围

将来还可能加部门或其它范围

不过Zhongjie使用的是字符串做层次，可以用  事件_类型_id 作为范围

由于要使用应用字符串参与到范围，所以BXJG.Utils.Application可以不做扩展


目前感觉最大的作用只是做 blazor server的事件总线，而它只需要 事件_级联标识，而这个可以定义在BXJG.RCL中
其它级别的可以暂时不做
但级联标记好像挺危险，比如每次搞个guid，那何时删除都不合适，而使用用户id，起码不会导致内存溢出

而其它场景现在没想那么细，暂时不用
