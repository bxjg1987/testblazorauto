

# blazor集成abp的实时通知
abp本身对signalR进行了集成，就是让我们更简单地使用signalR。
这属于底层的实时通信机制，我们目前没用它。

abp通知是另一套系统，其中基于signalR实现的实时通知接口IRealTimeNotifier只是其中一种实现方式，它依赖上面说的signalR


我们前端使用blazor，这里记录下本项目如何集成和使用实时通知  


后端是abp本身就集成好的，不用组任何变动，下面说说前端。、

前端参考signalR官方文档的.net客户端部分，

在BXJG.Utils.RCL的ServiceExt中，注册SignalR服务，其实就是定义HubConnection的工厂。
这里会做两个事：
- 订阅abp基本的消息通知事件，并通过zohngjie触发
- 处理前端事件（下面说），并通过zohngjie触发

然后在主项目的Router组件中，注入HubConnection，在启动时连接singalR，
并且在这里处理abp基本消息的显示，使用antblazor的消息组件。

>HubConnection本就是个类似事件总线的东东了，为什么还要用zhongjie触发一次？  
 主要目的是为了统一前端的编程方式，因为zhongjie是给整个前端界面用的事件总线，即使前端引用了abp，可能能
 能用它的事件总线，但最好别依赖它，毕竟我们是独立前端，依赖abp是迫不得已

到此，基本是实时消息显示通了。  
**<span style="color:red">通常后端可以通过abp的通知推送接口INotificationPublisher发送消息，通知推送接口会做持久化存储，默认已经会弹出消息通知了</span>**  
也可以通过实时通知接口，直接发送消息，这个不会持久化存储消息。

> 可以扩展其它类型的实时消息通知实现IRealTimeNotifier，这个只看官方文档即可。  
另外这里只是说实时通知，消息还可以单独管理，参考abp的通知文档。

# 前端事件
abp的事件是进程内的，不是分布式的。估计也是在内部存储委托。

上面说了，通过abp的通知发布器发送的消息会做持久化存储，它的目的是给用户发消息，
实时推送给当前在线用户只是后面的一个环节，而且推送方式除了网页实时还有其它方式。

有时候我们希望后端发生什么事情时，直接推送给前端，典型的例子是 后端配置变化了，
实时通知当前在线的用户。 注意范围，若只是某个租户的配置变化了，应该只推送这个租户下的所有由用户、
同理，若是某个具体用户的配置变化了，只通知这个用户关联的多个在线客户端（一个用户可能在多个端同时登录）

这种情况，不应该做持久化，只关注在线的用户即可。

这种情况最简单的是直接使用IRealTimeNotifier接口，或更具体的直接使用SignalRRealTimeNotifier。
但abp设计IRealTimeNotifier的目的是给指定用户发送消息，
```
public interface IRealTimeNotifier
{
    //
    // 摘要:
    //     If true, this real time notifier will be used for sending real time notifications
    //     when it is requested. Otherwise it will not be used.
    //
    //     If false, this realtime notifier will notify any notifications.
    bool UseOnlyIfRequestedAsTarget { get; }

    //
    // 摘要:
    //     This method tries to deliver real time notifications to specified users. If a
    //     user is not online, it should ignore him.
    Task SendNotificationsAsync(UserNotification[] userNotifications);
}
```
看参数名称也能看出来，

但在我们这里的需求是：给所有用户、给指定租户下的所有用户、给指定用户  发送事件消息。
为此，我们实现了UIEventTrigger，它在内部还是使用IRealTimeNotifier，但它实现了我们这里需求。

**<span style="color:red">只需要在有这种需求时，注入UIEventTrigger并调用它的方法，前端通过zohngjie注册事件处理逻辑</span>**
