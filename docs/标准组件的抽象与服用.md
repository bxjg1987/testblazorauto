# 标准组件
所谓的标准组件指：curd和选择组件，crud组件容易理解，选择组件指，在新增或编辑一个数据时，需要选择另一个数据，比如下订单时，需要选择客户，那么应该为选择客户的功能单独定义一个组件，这样整个项目都可以使用这个组件来选择客户了。为了下面方便描述，就把选择组件成为s吧（select）

# 抽象
大部分信息都需要crud和选择组件，而这些组件有很多相似之处，也有一些各自独特的点，分别为：c、r、u、s定义抽象组件，这样不同的信息针对cruds只需要提供相应的继承，然后添加一些特有的代码即可。
由于我们的应用服务层做了良好的curd接口抽象，因此实现起来比较容易。

由于修改和详情查看它俩密切相关，切需要无缝切换，所以把它俩定义在一个组件中。

目前的抽象仅仅包含逻辑部分，具体的功能需要自己定义UI，因为UI定制的可能性更大，不过逻辑部分也预留了很多虚方法可以重写。
UI部分的重用与定制化，组合比继承更容易，因为组合可以很容易使用razor标签来定制UI

在BXJG.Utils.RCL中已经实现了列表、新增、修改和详情，它与abp无关，因为需要在blazor auto模式的服务端和客户端模式共享，属于纯前端。
它与具体的项目无关，所有项目都引用它。

在ZLJ.RCL中定义了对应的子类，需要重写的部分很少。
此项目与具体的项目有关，但与具体的应用无关，同样，它也是纯前端，不依赖abp

具体的项目，如：ZLJ.Admin.CoreRCL中的crud组件，通常继承至ZLJ.RCL中的抽象组件，并自己提供UI


# 弹窗还是页面
在新增、查看详情、修改时可以弹窗来展示处理界面，也可以打开新页面来处理。个人觉得多tab布局时，弹窗更友好。
但打开新页面的好处是用户录入未完成时，可以切换到其它页面处理些别的事情，再回到刚才页面继续后续工作。
两种方式都行，我们决定统一使用弹窗方式，但也需要考虑将来换的情况。

# Modal弹窗组件
Modal是弹窗基础(它建立在Dialog基础上)，它是标准的模板化组件，在页面插入此组件时，弹窗头、体、底部都可以插入我们自己的组件。

上面分析考虑使用弹窗来展示新增、修改和详情，但也不确定将来是否会换成页面来展示，所以核心组件要和展现方式分离。
也就是定义新增、修改和详情组件时，不要引用Modal相关的东东，它是纯粹的组件，实现具体的功能，向外暴露方法、事件、属性等。

重点是问题是定制化，拿新增和详情组件来说，我们希望它的表单和保存按钮、和其它按钮在同一个组件中，因为它们密切相关。
但大部分的UI框架设计的弹窗都是有主题和Footer部分，它们是分离的，这就要求 把详情表单定义在一个组件中，把它关联的按钮定义在另一个组件中
这就比较尴尬了。

一种直接的办法是在核心组件中把按钮对应的处理定义成方法， 由弹窗组件来组合它和对应的按钮组件，按钮点击时调用对应的方法。
这就有点脱了裤子放屁的感觉。

幸亏net8的blazor中提供了section功能，简单点理解就是，我在外层组件的两个不同区域挖两个洞，让内层组件来填充这个两个洞，重点是填充这俩动的元素可以来自同一个子组件
（如果是来做不同的组件，那就仅仅是标准的模板化组件的功能）
>section参考：https://learn.microsoft.com/zh-cn/aspnet/core/blazor/components/sections?view=aspnetcore-8.0

前面分析了，弹窗内的组件和弹窗是分离的，弹窗内的组件是针对各种具体的业务的，外层的弹窗组件我们可以定义成通用的方式。
定义一个弹窗组件，泛型参数指定为 弹窗内组件的类型，通过动态组件展示即可。或也可以通过BuildTree方式来做。

需要考虑的是如何从列表页传递参数到弹窗内组件，动态组件已经支持了，若是BuildTree还需要研究下。
同理内层组件还需要反馈事件或结果到列表组件

# 高度抽象
上面分析了弹窗内组件和通用弹窗方案，其实可以针对具体的业务，封装高度封装的弹窗组件，比如 客户新增弹窗，它是结合客户新增组件和通用弹窗方案形成的组件。
这在后面要描述的选择组件会用到。比如选择客户时没找到，可以直接在选择那里点击新增 弹出客户新增弹窗出来。

不过除了选择时没有就新增的这种场景，好像没有其它地方需要服用此逻辑，貌似不是特别有必要封装。

# 选择组件
信息除了crud外，可能被其它地方选择，比如说：客户除了crud，在新增订单时需要选择客户。
我们已经定义了抽象的选择组件，它不仅仅是包含逻辑代码，因为它继承antblazor的选择组件，且它是泛型的。
所以其实可以算是通用组件，不过还是建议为特定的信息建立对应的选择组件，

最好用继承，这样最终的组件上保留所有select的功能，如果用组合，还得把内层组件的成员全部暴露一遍，太麻烦，也不合理。

上面高度抽象说明了，也许这里需要考虑没有选择项的时候，需要弹出新增弹窗。
