# 2024/4/8之前的描述
BXJG.Common.RCL作为公共的、与ABP无关的、与具体项目无关的、与具体UI无关的公共类库

它被服务端和客户端引用
它无法引用Microsfot.Aspnetcore.App，因为浏览器不支持

有些功能可能仅仅是服务端需要的，比如blazor server的电路
由于BXJG.Utils最初就没考虑客户端，所以现在就把Utils.RCL仅仅肯为服务端的公共库
而BXJG.Common看成服务端和客户端公用的公共库

另外Utils是与abp相关的，且包含一些公共的业务

不要定义在前后端共享，且与abp相关的 共享RCL，因为我们的目标是在前端不要引用abp相关功能
（但会引用abp库，因为包含一些dto和接口，但不会添加abp相关服务到容器和启动abp应用，因为老的abp未考虑blazor）

此库总结：
仅包含blazor server和auto模式中的服务端的相关功能，
在多个项目间共享
需要在前后端共享的不要使用此库，而是：若与abp有关则定义在ZLJ.Web.Blazor；若与abp无关的则放BXJG.Common.RCL中
